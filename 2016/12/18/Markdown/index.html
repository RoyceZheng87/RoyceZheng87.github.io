Markdown

Veröffentlicht am 2016-12-18

因为blog是用Hexo搭建的，所以使用的是Markdown，此前对于Markdown不太了解，借此机会学习下。

概述

Markdown的灵感来源于纯文本电子邮件的格式，目的就是实现[易读易写]。为此，Markdown的语法全部是由一些符号组成，这些符号经过精挑细选，其作用一目了然。

兼容HTML

Markdown的语法种类很少，只对应HTML标记的一小部分，HTML是一种发布的格式，Markdown是一种书写的格式。就这样，Markdown的格式语法只涵盖纯文本可以涵盖的范围。
不在Markdown涵盖范围之内的标签，都可以直接在文档里面用HTML撰写。不需要额外标注这是HTML或是Markdown，只要直接加标签就可以了。
要注意的只有一些HTML区块元素，比如：<div>、<table>、<pre>、<p>等标签，必须在前后加上空行与其他内容区分开，还要求它们的开始标签和结束标签不能用制表符或空格来缩进。而在HTML区块标签里的Markdown格式语法将不会被处理。
HTML的行内标签如<span>、<cite>、<del>可以在Markdown的段落，列表或是标题里随意使用，甚至可以不用Markdown格式，而直接采用HTML标签来格式化。Markdown语法在HTML行内标签间是有效的。

特殊字符自动转换

在HTML文件中，有两个字符需要特殊处理：<和&，如果你只是想要显示这些字符的原型，你必须要使用实体的形式，像是&lt;和&amp;
Markdown让你可以自然地书写字符，需要转换的由它自己处理。需要注意的是，code范围内，无论是行内还是区块，<和&两个字符都一定会被转换成HTML实体。

看到了这么多Markdown的特点，是不是已经迫不及待想知道如何使用它了，下面开始介绍正文。

区块元素

段落和换行

一个Markdown段落是由一个或多个连续的文本行组成，它的前后要有一个以上的空行，（这里的空行是指显示上像是空的就可以，比如说某一行只包含空格和制表符，则改行也会被视为空行）。

标题

Markdown支持两种标题的语法，类Setext和类atx形式。
类Setext形式是用底线的形式，利用=（最高阶标题）和-（第二阶标题），例如：

This is an H1
============
this is an H2
------------

两个以上数量的=和-都可以有效果。
类Atx形式则是在行首插入1到6个#，对应到标题1到6阶，例如：

# 这是 H1
## 这是 H2
###### 这是 H6

#号和后面的标题是有空格的。为了美观，你也可以在行尾处添加“闭合”类Atx样式的标题，在行尾加上#，而行为的数量并没有要求和开头一致。

区块引用 Blockquotes

Markdown标记区块引用是使用类似email中用>的引用方式。
只需要在引用行的最前面加上>，偷懒的情况下也可以在整个段落的第一行最前面加>。区块引用不仅可以嵌套，也允许在引用区块内使用其他Markdown语法。

> ## 这是一个标题。
> 
> 1.   这是第一行列表项。
> 2.   这是第二行列表项。
> 
>> 给出一些例子代码：
>> 
>> return shell_exec("echo $input | $markdown_script");

列表

Markdown支持有序列表和无序列表。
无序列表使用星号、加号或是减号作为列表标记。（3种方式等价，每种方式相当于重新起段落，不能混合使用）

* Red
* Green
* Blue

有序列表则使用数字接着一个英文句点。（列表中的数字顺序，及大小无所谓，哪怕都是一样的也可以使用，但不建议这样做）

1. Bird
1. McHale
1. Parish
3. Bird
1. McHale
8. Parish

列表项目标记通常时放在最左边，可以增加缩进，但最多3个空格，4个空格就相当于在列表项下另起一段，为了让列表好看，建议把内容用固定的缩进整理好。

* Lorem ipsum dolor sit amet, consectetuer adipiscing elit.
  Aliquam hendrerit mi posuere lectus. Vestibulum enim wisi,
  viverra nec, fringilla in, laoreet vitae, risus.
* Donec sit amet nisl. Aliquam semper ipsum sit amet velit.
  Suspendisse id sem consectetuer libero luctus adipiscing.

如果要在列表项目内放入引用，那>就需要缩进
如果要放代码区块的话，该区块就需要缩进两次，也就是8个空格或是2个制表符（在语法说明上看到这个，对于空格的个数不是很理解，本人认为只要一致就好）
对于1987. I'm born需要写成1987\. I'm born，否则会当成有序列表。

代码区块

在Markdown中，代码区块会用<pre>和<code>标签来表示，而你只需要简单的用4个空格缩进或是1个制表符即可。
注意，代码块需要和普通段落中间存在一个空行。一个代码块会一直持续到没有缩进的那一行（或是文件结尾）。代码块中的<和&会由Markdown来处理，当然代码块中的Markdown语法也不会被转换。

分隔线

在一行中使用三个以上的星号、减号、底线来建立一个分隔线，行内不能有其他内容。为了和类Setext标题写法区分开，建议大家在星号或者减号中间插入空格。

* * *

***

*****

- - -

---------------------------------------

_ _ _ _ _

区段元素

链接

Markdown支持行内式和参考式两种形式的链接语法，不管是哪一种，链接文字都是用[方括号]来标记。
行内式链接，只要在方括号后面紧接着圆括号并插入网址链接即可，也可用双引号加入链接文字。例如：

This is [an example](http://example.com/ "Title") inline link.

See my [About](/about/) page for details.

参考式链接，是在链接文字的方括号后再加另一个方括号，在第二个方括号中填入用以辨识链接的标记，接着，在文件的任意处，添加此标记的链接内容定义出来：

This is [an example][id] reference-style link.     

[id]: http://example.com/  "Optional Title Here"

注意，链接标识不区分大小写，另外建议链接title使用双引号包起来。
我们也可以隐藏链接标记，省略标记，链接标记会是为等同于链接文字。链接的定义可以放在文件中的任何一个地方，建议放在最后，相当于注解一样。使用参考式链接，可以让文件更像是浏览器最后产生的结果，让你可以把一些标记相关的元数据移到段落文字之外，从而不会打断阅读。

I get 10 times more traffic from [Google][] than from
[Yahoo][] or [MSN][].

  [google]: http://google.com/        "Google"
  [yahoo]:  http://search.yahoo.com/  "Yahoo Search"
  [msn]:    http://search.msn.com/    "MSN Search"

强调

Markdown使用星号*和底线_作为标记强调的符号，会被转换成<em>标签。用两个*和_，则会转换为<strong>，例如：

*single asterisks*

_single underscores_

**double asterisks**

__double underscores__

注意，如果要插入普通的*和_，你可以用反斜线：

\*this text is surrounded by literal asterisks\*

代码

如果要标记一小段行内代码，可以用反引号(` )，例如：

Use the `printf()` function.

如果要在代码区段内插入反引号，用多个反引号来开启和结束代码，起始和末尾的反引号各加一个空格。

`` There is a literal backtick (`) here. ``

A single backtick in a code span: `` ` ``

A backtick-delimited string in a code span: `` `foo` ``

图片

图片和链接一样，分为行内式和参考式
行内式，一个惊叹号(!)，接着方括号中加入图片的替代文字，接着圆括号中是图片的网址，最后还可以用引号包住并加上选择性的“title”文字，例如：

![Alt text](/path/to/img.jpg)

![Alt text](/path/to/img.jpg "Optional title")

参考式，一个惊叹号(!)，接着其他和链接的参考式一样。

![Alt text][id]

[id]: url/to/image  "Optional title attribute"

其他

反斜杠

Markdown 支持以下这些符号前面加上反斜杠来帮助插入普通的符号：

\   反斜线
`   反引号
*   星号
_   底线
{}  花括号
[]  方括号
()  括弧
#   井字号
+   加号
-   减号
.   英文句点
!   惊叹号
